TripleWave

An open-source tool to bring RDF streams on the Web

Download this project as a .zip file Download this project as a tar.gz file

TripleWave is a framework to publish RDF streams on the Web. It generates RDF streams out of existing Web streams (e.g., Wikipedia change and Twitter streams) or RDF datasets containing some time-related annotations.

Installation

Requirements

  • nodejs (tested on version 4.4.0 and 5)
  • java 8 (for fuseki, theoretically you can ignore this if you plan to use only the transform mode)

In order to install TripleWave just clone the repository

git clone https://github.com/streamreasoning/TripleWave.git

Then simply run

npm install

for installing the node depdencies.

How to run

To start TripleWave, run the start.sh file if you are using Mac/Linux, or start.bat if you are using Windows.

By default, TripleWave starts on the ports 8114 (stream description) and 8124 (stream distribution), and it converts the English Wikipedia changes steram. To customise the TripleWave behaviour, see the Configuration section.

Configuration

Configuration file

The configuration file is located in config folder allows to fully customize the TripleWave installation.

Modes

TripleWave can work on three different modes, depending on input source and the desired output.

Transform

Replay

The Replay mode converts an existing dataset (containing some temporal information) in an RDF stream and streams it out. In the following, we explain how to configure TripleWave in order to work in this mode.

Set the input file

The user has to set the file location of the RDF file to be converted. It can be done by setting the field rdf_file in the config file, 

rdf_file=../rdf/data.ttl

The conversion happens through a set of SPARQL queries that are ran from TripleWave
